## О проекте
Данный проект представляет собой Java-приложение для вычисления сложной кусочно-заданной математической функции. Главная ценность проекта заключается не только в реализации математических алгоритмов (через разложение в ряды Тейлора), но и в демонстрации продвинутых подходов к архитектуре и обеспечению качества кода.

Проект построен по принципам модульности, где сложные функции компонуются из базовых, образуя строгую иерархию зависимостей.

### Математическая модель системы

Программа реализует вычисление кусочно-заданной функции $f(x)$ с заданной точностью $\varepsilon$. Для базовых тригонометрических и логарифмических операций используется разложение в ряд Тейлора:

$$
f(x) = 
\begin{cases} 
\left(\left(\left((\cos(x) - \csc(x) + \cos(x) + \sin(x))^3\right)^2\right)^3\right)^2 - (\sec(x) \cdot \tan(x) \cdot \sin(x) - 2\cos(x)), & \text{если } x \le 0 \\
\frac{((\ln(x) - \log_{10}(x) + \log_2(x)) \cdot \ln(x)) \cdot \log_3(x)}{\frac{\log_2^3(x)}{\log_5^3(x)}}, & \text{если } x > 0 
\end{cases}
$$

## Архитектура системы
Вычисления разделены на независимые слои (уровни), что обеспечивает слабую связность (loose coupling) и высокую тестируемость:
* **Уровень 0 (Базовый):** Вычисление `sin(x)` и `ln(x)` с использованием разложения в ряд Тейлора с заданной точностью.
* **Уровень 1 и 2:** Производные функции: `cos(x)`, `tan(x)`, `sec(x)`, `csc(x)`, а также логарифмы по заданному основанию (`log_2`, `log_3`, `log_5`, `log_10`).
* **Уровень 3:** Группировка в функции левой ($x \le 0$) и правой ($x > 0$) ветвей уравнения.
* **Уровень Системы:** Финальный класс `SystemOfFunctions`, управляющий потоком выполнения и переключением ветвей.

## Стратегия тестирования (Bottom-Up Integration)
В проекте реализован комплексный подход к тестированию, имитирующий реальные процессы энтерпрайз-разработки:

1. **Data-Driven Testing (DDT):** Все ожидаемые результаты заранее рассчитаны и хранятся во внешних `.csv` файлах. Фреймворк **JUnit 5** (`@ParameterizedTest`, `@CsvFileSource`) автоматически подгружает данные, прогоняя систему через сотни контрольных точек.
2. **Восходящая интеграция (Bottom-Up):** Интеграционные тесты спроектированы пошагово. На начальных этапах зависимости заменяются на заглушки с помощью **Mockito**, затем они постепенно заменяются на реальные объекты (от Уровня 0 до Уровня Системы). Это позволяет локализовать ошибки на стыках модулей.
3. **Обработка граничных условий:** Тесты строго валидируют поведение системы в точках разрыва (асимптоты) и при делении на ноль, проверяя корректный выброс `ArithmeticException` и обрабатывая `NaN`.
4. **Динамическая погрешность:** Реализован алгоритм плавающей погрешности (delta) для безопасного сравнения чисел с плавающей точкой, достигающих гигантских порядков.
